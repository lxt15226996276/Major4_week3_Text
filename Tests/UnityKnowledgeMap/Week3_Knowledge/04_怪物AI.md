## 一、怪物 AI 架构（官方标准 · 必背）

【感知 Perception】→ 【决策 Decision / FSM】→ 【行动 Action】→ 【动画 Animation】

## 二、行为状态定义（⭐⭐⭐ · Lab 标准）

### 2.1 四态 + 死亡（Lab_AI_01）

| 状态 | 进入条件 | 行为 | 频率 |
|------|----------|------|------|
| **Patrol** | 默认；脱战完成 | A↔B 巡逻或 MoveTowards | ⭐⭐⭐ |
| **Chase** | 与玩家距离 < `chaseRange` | 朝玩家移动 | ⭐⭐⭐ |
| **Attack** | 距离 < `attackRange` | 停移 + 攻击 + 冷却 | ⭐⭐⭐ |
| **Return** | 距离 > `loseRange`（P+1） | 回最近巡逻点 | ⭐⭐ |
| **Die** | HP ≤ 0 | 播死亡动画 + 禁用 AI | ⭐⭐⭐ |

### 2.2 状态转移图

```
                    ┌──────────────────────────────────┐
                    │                                  │
                    ↓                                  │
              ┌─────────┐  d < chaseRange   ┌─────────┐
              │ Patrol  │──────────────────→│  Chase  │
              └─────────┘←──────────────────└─────────┘
                    ↑    d > loseRange          │
                    │    (Return 完成)          │ d < attackRange
                    │                           ↓
              ┌─────────┐                   ┌─────────┐
              │ Return  │←── d > loseRange ─│ Attack  │
              └─────────┘                   └─────────┘
                    │                           │
                    └──── 回到 Patrol ──────────┘
                              (冷却结束且仍近 → 再 Attack)

任意状态 ── HP ≤ 0 ──→ Die
```

### 2.3 距离参数设计（必背）

| 参数 | 典型值 | 关系 |
|------|--------|------|
| `attackRange` | 2m | 最小 |
| `chaseRange` | 8m | > attackRange |
| `loseRange` | 12m | **> chaseRange** |

**[重难点·Lab 面试]** 脱战距离为什么要大于追击距离？
→ **迟滞（Hysteresis）**：若 chaseRange == loseRange，边界上来回抖动会导致 Patrol ↔ Chase 每帧切换。loseRange 更大，玩家跑出更远才脱战。

**[易错点]**
- `chaseRange < attackRange` → 永远进不了 Attack
- `loseRange < chaseRange` → 刚追就脱战

## 三、感知系统（考纲 180 · ⭐⭐⭐）

### 3.1 距离检测（Lab 默认）

```csharp
Transform player;
public float chaseRange = 8f;
public float attackRange = 2f;
public float loseRange = 12f;

void Awake()
{
    var go = GameObject.FindWithTag("Player");  // 仅 Awake 一次
    if (go != null) player = go.transform;
}

float DistanceToPlayer()
{
    return Vector3.Distance(transform.position, player.position);
}

// P+1：sqrMagnitude 更快，无开方
float SqrDistanceToPlayer()
{
    return (player.position - transform.position).sqrMagnitude;
}

bool InChaseRange() => SqrDistanceToPlayer() < chaseRange * chaseRange;
```

| 方式 | 说明 | 频率 |
|------|------|------|
| `Vector3.Distance` | 直观，含开方 | ⭐⭐⭐ |
| `sqrMagnitude` 比较 | 与 range² 比，性能更好 | ⭐⭐ |
| **球形 Trigger** | OnTriggerEnter/Stay 感知 | ⭐⭐ |
| 每 0.2s 检测 | 降低 CPU，AI 可接受 | ⭐⭐ |

**[教材精华·Lab]** 不要 `Find("Player")` 每帧；**Awake 缓存** `Transform player`。

### 3.2 球形 Trigger 感知（备选）

| 步骤 | 操作 |
|:---:|------|
| 1 | 子物体加 **Sphere Collider**，勾选 **Is Trigger** |
| 2 | 半径 = chaseRange |
| 3 | OnTriggerEnter → 记录 player 在范围内 |

**[对比]** Distance 实现简单；Trigger 适合「进入区域才激活」的优化。

## 四、Patrol 巡逻（考纲 177 · ⭐⭐⭐）

### 4.1 两点巡逻 A↔B

```csharp
public Transform pointA;
public Transform pointB;
Transform currentTarget;
float waitTimer;
public float patrolWaitMin = 1f;
public float patrolWaitMax = 3f;
bool isWaiting;

void EnterPatrol()
{
    currentTarget = pointA;
    isWaiting = false;
}

void UpdatePatrol()
{
    if (InChaseRange()) { ChangeState(EnemyState.Chase); return; }

    if (isWaiting)
    {
        waitTimer -= Time.deltaTime;
        if (waitTimer <= 0f) isWaiting = false;
        return;
    }

    transform.position = Vector3.MoveTowards(
        transform.position, currentTarget.position, patrolSpeed * Time.deltaTime);

    if (Vector3.Distance(transform.position, currentTarget.position) < 0.2f)
    {
        currentTarget = currentTarget == pointA ? pointB : pointA;
        isWaiting = true;
        waitTimer = Random.Range(patrolWaitMin, patrolWaitMax);  // ⭐⭐⭐
    }
}
```

**[考纲 177]** 随机巡逻等待 → `Random.Range(patrolWaitMin, patrolWaitMax)`。

### 4.2 巡逻点 setup

| 方式 | 说明 |
|------|------|
| 空物体 A、B | Lab 推荐，可视化 |
| 数组 Transform[] | 多点随机 |
| 出生点 fallback | Return 时回到最近点 |

## 五、Chase 追击（⭐⭐⭐）

```csharp
void UpdateChase()
{
    float dist = DistanceToPlayer();

    if (dist > loseRange)
    {
        ChangeState(EnemyState.Return);  // 或直切 Patrol
        return;
    }
    if (dist < attackRange)
    {
        ChangeState(EnemyState.Attack);
        return;
    }

    // 方案 A：直移（Lab 默认）
    Vector3 dir = (player.position - transform.position).normalized;
    dir.y = 0f;
    transform.position += dir * chaseSpeed * Time.deltaTime;

    // 可选：朝向玩家
    if (dir.sqrMagnitude > 0.001f)
        transform.rotation = Quaternion.LookRotation(dir);
}
```

### 5.1 直移 vs NavMeshAgent（Lab 面试）

| 方案 | 优点 | 缺点 |
|------|------|------|
| **MoveTowards / 直移** | 简单、Lab 零配置 | **穿墙**、不绕障碍 |
| **NavMeshAgent.SetDestination** | 寻路、防穿墙、绕障碍 | 需 Bake NavMesh、配置 Agent |

```csharp
// 方案 B：NavMesh（P+1 / 工程）
NavMeshAgent agent;

void UpdateChaseNav()
{
    agent.SetDestination(player.position);
    if (DistanceToPlayer() < attackRange)
    {
        agent.isStopped = true;
        ChangeState(EnemyState.Attack);
    }
}
```

**[考纲 183]** 防穿墙 → NavMeshAgent 或碰撞后重算路径。

**[Lab 面试]** 追击用每帧 SetDestination（NavMesh）和直移各什么优缺点？→ 见上表。

## 六、Attack 攻击（考纲 179 · ⭐⭐⭐）

```csharp
float attackCooldown = 1.5f;
float lastAttackTime = -999f;
public int damage = 10;

void UpdateAttack()
{
    float dist = DistanceToPlayer();

    if (dist > attackRange)
    {
        ChangeState(EnemyState.Chase);
        return;
    }

    // 面向玩家，不位移
    Vector3 dir = player.position - transform.position;
    dir.y = 0f;
    if (dir.sqrMagnitude > 0.001f)
        transform.rotation = Quaternion.LookRotation(dir.normalized);

    if (Time.time - lastAttackTime >= attackCooldown)
    {
        lastAttackTime = Time.time;
        PerformAttack();
    }
}

void PerformAttack()
{
    animator.SetTrigger("Attack");
    // 伤害：Animation Event 帧 或 直接 ApplyDamage
    Debug.Log("Attack!");
    // playerHealth.TakeDamage(damage);
}
```

**[考纲 179]** 距离 < attackRange **且** 冷却结束 → 播动画 + 判定伤害。

| 要点 | 说明 |
|------|------|
| **attackCooldown** | 防每帧扣血 |
| **停移** | Attack 状态不调用 Move |
| **动画 Event** | 伤害帧精确同步（Week2 知识） |
| **玩家走远** | 切回 Chase |

## 七、Return 脱战（P+1 · ⭐⭐）

```csharp
Transform returnPoint;

void EnterReturn()
{
    returnPoint = GetNearestPatrolPoint();
}

void UpdateReturn()
{
    transform.position = Vector3.MoveTowards(
        transform.position, returnPoint.position, patrolSpeed * Time.deltaTime);

    if (Vector3.Distance(transform.position, returnPoint.position) < 0.3f)
        ChangeState(EnemyState.Patrol);

    // 若玩家又靠近，可重新 Chase
    if (InChaseRange())
        ChangeState(EnemyState.Chase);
}
```

**[教材精华·Lab]** Return 完成 → Patrol；脱战距离 > 追击距离，避免边界抖动。

## 八、完整 AI 状态机代码框架（⭐⭐⭐）

```csharp
public enum EnemyState { Patrol, Chase, Attack, Return, Die }

public class EnemyAI : MonoBehaviour
{
    EnemyState currentState = EnemyState.Patrol;
    [SerializeField] Animator animator;
    // ... 距离参数、引用、冷却 ...

    void Awake()
    {
        // 缓存 player、agent、animator
    }

    void Update()
    {
        if (currentState == EnemyState.Die) return;

        if (hp <= 0) { ChangeState(EnemyState.Die); return; }

        switch (currentState)
        {
            case EnemyState.Patrol:  UpdatePatrol();  break;
            case EnemyState.Chase:   UpdateChase();   break;
            case EnemyState.Attack:  UpdateAttack();  break;
            case EnemyState.Return:  UpdateReturn();  break;
        }

        SyncAnimator();
    }

    void ChangeState(EnemyState next)
    {
        if (currentState == next) return;
        OnExitState(currentState);
        currentState = next;
        OnEnterState(next);
    }

    void SyncAnimator()
    {
        bool isMoving = currentState == EnemyState.Chase
                     || currentState == EnemyState.Patrol
                     || currentState == EnemyState.Return;
        animator.SetBool("IsWalking", isMoving);
    }
}
```

**[重难点]** 与 [03_状态机.md](03_状态机.md) 同一套：`ChangeState` + Enter/Exit + 全局 Die。

## 九、怪物动画与行为联动（考纲 176、182 · ⭐⭐⭐）

### 9.1 基本动画状态

| 行为状态 | Animator 参数 | 动画 |
|----------|--------------|------|
| Patrol / 站立 | — | Idle |
| Chase / Patrol 移动 | `SetBool("IsWalking", true)` | Walk |
| Attack | `SetTrigger("Attack")` | Attack |
| Die | `SetTrigger("Die")` | Die |

**[考纲 176]** 脚本读**行为状态**，设 **Animator 参数**切动画——不是 Animator 反过来驱动 AI 逻辑。

### 9.2 Exam04 Enemy.controller

| 项目 | 说明 |
|------|------|
| Prefab | `Exam04/Prefabs/Enemy.prefab` 已挂 Controller |
| 第 4 步 | Default State = **Idle**，Instantiate 即播 |
| 第 5 步 | 受击 **`Die` Trigger** + `GameManager.OnEnemyDied()` |

```csharp
void EnterDie()
{
    animator.SetTrigger("Die");
    GetComponent<Collider>().enabled = false;
    this.enabled = false;  // 停 AI Update
    Destroy(gameObject, 2f);
}
```

**[易错点]** Die 后脚本仍跑 Chase → 必须 disable AI 或 early return。

## 十、死亡与 GameManager 回调（Exam04 · ⭐⭐⭐）

```csharp
// EnemyController · Exam04 递进
public class EnemyController : MonoBehaviour
{
    [SerializeField] int hp = 3;
    GameManager gameManager;

    public void Init(GameManager gm) => gameManager = gm;

    void OnTriggerEnter(Collider other)
    {
        if (!other.CompareTag("Bullet")) return;
        Destroy(other.gameObject);
        hp--;
        if (hp <= 0)
        {
            gameManager?.OnEnemyDied();
            Destroy(gameObject);
        }
    }
}
```

| 要点 | 说明 |
|------|------|
| **Init 注入** | 禁 Find GameManager |
| **aliveCount--** | GameManager 统计全灭胜利 |
| **回调** | 敌人不查 UI，只通知 Manager |

**[教材精华·吴亚峰/Exam04]** 胜利 = **敌人全灭**，不是 Exam03 加血满条。

## 十四、注意要点与最佳实践

### 常见陷阱

| 易错点 | 说明 |
|--------|------|
| Find 每帧 | Awake 缓存 player |
| 距离参数乱设 | attack < chase < lose |
| 攻击无 CD | 玩家秒死 |
| Exam04 第 4 步写追击 | 卷面只要 Idle |
| AI 与动画各一套 | 行为 FSM 为主，Animator 跟随 |
| 直移穿墙 | Lab 可接受；工程用 NavMesh |
| Die 后仍 Update | disable 脚本 |

### 最佳实践

- 先画转移表（见 §2.2），再写 switch
- 距离比较优先 sqrMagnitude
- 感知/决策/行动/动画四层分离
- Attack 伤害放 Animation Event 或冷却后单次
- GameManager 回调，不 Find
- Lab 验收：1 怪四态肉眼可演示 + 可跑脱战

## 十八、教材 vs 官方 · 对照速查

| 知识点 | 教材常见写法 | 工程推荐 |
|--------|--------------|----------|
| 找玩家 | Find 每帧 | **Awake 缓存 Tag** |
| 追击 | 写死 if-else | **FSM switch** |
| 脱战 | 同距离进出 | **loseRange > chaseRange** |
| 攻击 | 每帧伤害 | **cooldown + Trigger** |
| 动画 | Play("Walk") | **SetBool/SetTrigger** |
| 穿墙 | 直移 | **NavMeshAgent** |
| 胜利 | 加血满 | **敌人全灭计数** |
