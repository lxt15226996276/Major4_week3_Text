## 一、编辑器扩展核心概念（官方标准 · 必背）

```
Unity 编辑器架构
├── Runtime 层（打包进游戏）
│   └── UnityEngine 命名空间
│       ├── MonoBehaviour
│       ├── GameObject
│       └── 运行时组件
│
└── Editor 层（仅编辑器环境）
    └── UnityEditor 命名空间
        ├── MenuItem
        ├── EditorWindow
        ├── CustomEditor
        ├── PropertyDrawer
        └── AssetDatabase
```

**[重难点]** 编辑器脚本**必须**放在名为 `Editor` 的文件夹中（或其子目录），否则打包时会报错——因为 `UnityEditor` 命名空间在运行时不可用。

**[易错点]** Exam01 编辑器脚本放 `Assets/Exams/Exam01/Scripts/Editor/` 是正确的；放 `Assets/Exams/Exam01/Scripts/` 会导致打包失败。

## 二、MenuItem 菜单扩展（⭐⭐⭐）

### 2.1 基础用法

| 项 | 说明 |
|----|------|
| **是什么** | 在 Unity 顶部菜单栏添加自定义菜单项 |
| **命名空间** | `UnityEditor` |
| **路径规范** | `一级菜单/二级菜单/三级菜单` |

```csharp
using UnityEditor;

public static class MyMenuItems
{
    [MenuItem("我的工具/批量重命名")]
    public static void BatchRename()
    {
        Debug.Log("批量重命名工具");
    }
}
```

### 2.2 菜单优先级

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `priority` | 菜单显示顺序，值越小越靠前 | 1000 |

```csharp
[MenuItem("我的工具/场景搭建", priority = 10)]
public static void BuildScene() { }

[MenuItem("我的工具/资源清理", priority = 20)]
public static void CleanAssets() { }
```

### 2.3 快捷键

| 快捷键 | 写法 |
|--------|------|
| **Ctrl/Cmd + A** | `%a` |
| **Shift + A** | `#a` |
| **Alt + A** | `&a` |
| **组合** | `%#&a`（Ctrl+Shift+Alt+A） |

```csharp
[MenuItem("我的工具/快速保存 %s")]
public static void QuickSave()
{
    EditorSceneManager.SaveCurrentModifiedScenesIfUserWantsTo();
}
```

### 2.4 验证方法

```csharp
[MenuItem("我的工具/删除选中物体")]
public static void DeleteSelected()
{
    foreach (var obj in Selection.gameObjects)
        Object.DestroyImmediate(obj);
}

[MenuItem("我的工具/删除选中物体", true)]
public static bool ValidateDeleteSelected()
{
    return Selection.gameObjects.Length > 0;
}
```

**[教材精华·杜亚南/王磊]** 验证方法返回 `false` 时，菜单项变灰不可用；用于防止误操作。

**[易错点]** 验证方法签名必须与实际方法**完全一致**，仅多 `true` 参数。

## 三、EditorWindow 自定义窗口（⭐⭐⭐）

### 3.1 窗口生命周期

| 回调 | 时机 | 典型用途 |
|------|------|----------|
| **OnEnable** | 窗口创建/激活 | 初始化、加载配置、订阅事件 |
| **OnDisable** | 窗口销毁/失活 | 保存配置、取消订阅、清理资源 |
| **OnGUI** | 每帧绘制界面 | 绘制按钮、输入框、布局 |
| **Update** | 每帧更新（可选） | 实时更新数据、动画 |

```csharp
using UnityEditor;
using UnityEngine;

public class MyToolWindow : EditorWindow
{
    string inputText = "";
    int counter = 0;

    [MenuItem("我的工具/自定义窗口")]
    public static void ShowWindow()
    {
        GetWindow<MyToolWindow>("我的工具");
    }

    void OnEnable()
    {
        titleContent = new GUIContent("我的工具");
    }

    void OnGUI()
    {
        GUILayout.Label("基本设置", EditorStyles.boldLabel);
        
        inputText = EditorGUILayout.TextField("输入文本", inputText);
        
        if (GUILayout.Button("点击计数"))
            counter++;
        
        EditorGUILayout.LabelField("计数", counter.ToString());
    }
}
```

### 3.2 窗口布局

| 布局方式 | 说明 | 频率 |
|----------|------|------|
| **EditorGUILayout** | 自动布局，自适应窗口大小 | ⭐⭐⭐ |
| **GUILayout** | 自动布局（通用） | ⭐⭐ |
| **GUI** | 固定位置布局（不推荐） | ⭐ |

```csharp
void OnGUI()
{
    using (new EditorGUILayout.HorizontalScope())
    {
        GUILayout.Button("左");
        GUILayout.Button("右");
    }
    
    using (new EditorGUILayout.VerticalScope("Box"))
    {
        EditorGUILayout.LabelField("内容区域");
        EditorGUILayout.TextField("输入");
    }
}
```

**[教材精华·宣雨松]** 编辑器窗口优先用 `EditorGUILayout`，提供更好的 Unity 风格和自动布局。

**[易错点]** 在 EditorWindow 的 OnGUI 中用 `GUILayout` 而非 `GUI`，避免手动计算坐标。

## 四、CustomEditor 自定义 Inspector（⭐⭐⭐）

### 4.1 基础实现

```csharp
using UnityEditor;
using UnityEngine;

[CustomEditor(typeof(MyComponent))]
public class MyComponentEditor : Editor
{
    public override void OnInspectorGUI()
    {
        DrawDefaultInspector();
        
        MyComponent targetScript = (MyComponent)target;
        
        if (GUILayout.Button("执行方法"))
            targetScript.DoSomething();
    }
}
```

### 4.2 SerializedObject 与 SerializedProperty

| 类 | 作用 | 频率 |
|----|------|------|
| **SerializedObject** | 包装目标对象，支持序列化 | ⭐⭐⭐ |
| **SerializedProperty** | 包装单个属性，支持 undo/redo | ⭐⭐⭐ |

```csharp
using UnityEditor;
using UnityEngine;

[CustomEditor(typeof(PlayerStats))]
public class PlayerStatsEditor : Editor
{
    SerializedProperty health;
    SerializedProperty maxHealth;
    SerializedProperty attack;

    void OnEnable()
    {
        health = serializedObject.FindProperty("_health");
        maxHealth = serializedObject.FindProperty("_maxHealth");
        attack = serializedObject.FindProperty("_attack");
    }

    public override void OnInspectorGUI()
    {
        serializedObject.Update();
        
        EditorGUILayout.PropertyField(health);
        EditorGUILayout.PropertyField(maxHealth);
        
        EditorGUILayout.Space();
        EditorGUILayout.PropertyField(attack);
        
        serializedObject.ApplyModifiedProperties();
    }
}
```

**[重难点]** `serializedObject.Update()` 和 `serializedObject.ApplyModifiedProperties()` 必须成对使用，否则修改不生效或不支持 undo。

**[易错点]** 直接修改 `target` 对象的字段不支持 undo；必须通过 `SerializedProperty`。

### 4.3 高级布局

```csharp
public override void OnInspectorGUI()
{
    serializedObject.Update();
    
    EditorGUILayout.BeginVertical("HelpBox");
    EditorGUILayout.LabelField("生命值", EditorStyles.boldLabel);
    EditorGUILayout.PropertyField(health);
    EditorGUILayout.PropertyField(maxHealth);
    EditorGUILayout.EndVertical();
    
    EditorGUILayout.Space();
    
    EditorGUILayout.BeginHorizontal();
    GUILayout.FlexibleSpace();
    if (GUILayout.Button("满血", GUILayout.Width(80)))
    {
        health.intValue = maxHealth.intValue;
    }
    GUILayout.FlexibleSpace();
    EditorGUILayout.EndHorizontal();
    
    serializedObject.ApplyModifiedProperties();
}
```

### 5.1 自定义属性标记

public class MinMaxRangeAttribute : PropertyAttribute

### 5.2 属性绘制器

```csharp
using UnityEditor;
using UnityEngine;

[CustomPropertyDrawer(typeof(MinMaxRangeAttribute))]
public class MinMaxRangeDrawer : PropertyDrawer
{
    public override void OnGUI(Rect position, SerializedProperty property, GUIContent label)
    {
        MinMaxRangeAttribute attr = (MinMaxRangeAttribute)attribute;
        
        EditorGUILayout.BeginHorizontal();
        EditorGUILayout.PropertyField(property.FindPropertyRelative("min"));
        EditorGUILayout.LabelField("～");
        EditorGUILayout.PropertyField(property.FindPropertyRelative("max"));
        EditorGUILayout.EndHorizontal();
    }
}
```

### 5.3 使用自定义属性

```csharp
public class Spawner : MonoBehaviour
{

    public Vector2 spawnRange;
}
```

**[教材精华·马遥]** PropertyDrawer 让你自定义任意字段类型的 Inspector 显示方式。

## 六、AssetDatabase 资源操作（⭐⭐⭐）

### 6.1 资源路径规范

| 路径类型 | 示例 | 频率 |
|----------|------|------|
| **相对路径** | `Assets/Exams/Exam01/Scenes/Exam01_MainScene.unity` | ⭐⭐⭐ |
| **绝对路径** | `D:/Project/UnityProject/Assets/...` | ⭐ |

### 6.2 常用 API

| API | 说明 | 频率 |
|-----|------|------|
| `AssetDatabase.CreateAsset(Object, string)` | 创建资源文件 | ⭐⭐⭐ |
| `AssetDatabase.DeleteAsset(string)` | 删除资源 | ⭐⭐⭐ |
| `AssetDatabase.MoveAsset(string, string)` | 移动资源 | ⭐⭐ |
| `AssetDatabase.Refresh()` | 刷新数据库 | ⭐⭐⭐ |
| `AssetDatabase.FindAssets(string)` | 搜索资源 | ⭐⭐ |
| `AssetDatabase.LoadAssetAtPath<T>(string)` | 加载资源 | ⭐⭐⭐ |
| `AssetDatabase.GetAssetPath(Object)` | 获取资源路径 | ⭐⭐ |

```csharp
[MenuItem("我的工具/创建场景")]
public static void CreateScene()
{
    string path = "Assets/MyScene.unity";
    
    if (File.Exists(path))
    {
        if (!EditorUtility.DisplayDialog("覆盖", "场景已存在，是否覆盖？", "覆盖", "取消"))
            return;
        AssetDatabase.DeleteAsset(path);
    }
    
    EditorSceneManager.SaveScene(
        EditorSceneManager.NewScene(NewSceneSetup.DefaultGameObjects), 
        path
    );
    
    AssetDatabase.Refresh();
    EditorUtility.DisplayDialog("完成", "场景创建成功！", "确定");
}
```

**[重难点]** 创建/删除资源后必须调用 `AssetDatabase.Refresh()`，否则 Project 窗口不更新。

**[易错点]** `File.Exists` 检查的是文件系统路径；`AssetDatabase.LoadAssetAtPath` 用的是相对路径。

| 方法 | 说明 | 频率 |

| `DisplayDialog(string, string, string, string)` | 显示对话框 | ⭐⭐⭐ |

| `ClearProgressBar()` | 清除进度条 | ⭐⭐⭐ |

```csharp
[MenuItem("我的工具/批量操作")]
public static void BatchOperation()
{
    int total = 100;
    
    for (int i = 0; i <= total; i++)
    {
        EditorUtility.DisplayProgressBar(
            "批量操作",

}
```

## 八、场景自动搭建实战（⭐⭐⭐）

```csharp
using UnityEditor;
using UnityEngine;
using UnityEngine.UI;
using UnityEditor.SceneManagement;

public static class SceneBuilder
{
    [MenuItem("工具/搭建主场景")]
    public static void BuildMainScene()
    {
        EnsureDirectory("Assets/Scenes");
        
        var scene = EditorSceneManager.NewScene(NewSceneSetup.DefaultGameObjects);
        
        GameObject canvas = new GameObject("Canvas");
        Canvas canvasComp = canvas.AddComponent<Canvas>();
        canvasComp.renderMode = RenderMode.ScreenSpaceOverlay;
        canvas.AddComponent<CanvasScaler>();
        canvas.AddComponent<GraphicRaycaster>();
        
        GameObject bg = new GameObject("Background");
        bg.transform.SetParent(canvas.transform);
        bg.AddComponent<Image>().color = new Color(0.1f, 0.1f, 0.1f);
        
        EditorSceneManager.SaveScene(scene, "Assets/Scenes/MainScene.unity");
        AssetDatabase.Refresh();
        
        Debug.Log("场景搭建完成！");
    }

    private static void EnsureDirectory(string path)
    {
        string[] parts = path.Split('/');
        string current = "";
        for (int i = 0; i < parts.Length; i++)
        {
            current = i == 0 ? parts[i] : current + "/" + parts[i];
            if (!AssetDatabase.IsValidFolder(current))
            {
                string parent = i == 0 ? "Assets" : string.Join("/", parts, 0, i);
                AssetDatabase.CreateFolder(parent, parts[i]);
            }
        }
    }
}
```

**[教材精华·吴亚峰/陈俊宇]** 场景搭建脚本是编辑器扩展最实用的场景之一，可大幅减少重复操作。

## 十、注意要点与最佳实践

### 常见陷阱

| 易错点 | 说明 |
|--------|------|
| 编辑器脚本放非 Editor 文件夹 | 打包报错 |
| 直接修改 target 对象 | 不支持 undo/redo |
| 忘记调用 serializedObject.Update/Apply | 修改不生效 |
| 每帧调用 AssetDatabase.Refresh | 性能极差 |
| 使用 GUI 而非 EditorGUILayout | 布局不统一 |
| 不处理 EditorApplication.isPlaying | Play 模式下误操作 |
| MenuItem 不设优先级 | 菜单顺序混乱 |
| EditorWindow 不处理 OnEnable/OnDisable | 资源泄漏 |

### 最佳实践

- 编辑器脚本**必须**放在 `Editor` 文件夹
- 修改对象用 `SerializedObject`/`SerializedProperty`，支持 undo
- 布局用 `EditorGUILayout`，自动适配
- 批量操作后调用 `AssetDatabase.Refresh()`
- 菜单加验证方法，防止误操作
- EditorWindow 生命周期正确处理
- 复杂工具用 `EditorUtility.DisplayProgressBar` 显示进度
