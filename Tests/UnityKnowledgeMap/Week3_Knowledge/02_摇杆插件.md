## 一、虚拟摇杆与输入分层（官方标准 · 必背）

```
【输入层】                    【逻辑层】                 【移动层】
EasyTouch Joystick    →    Vector2 direction    →    CharacterController.Move
键盘 Input.GetAxis    →    （统一接口）          →    或 transform 位移
自写 UGUI 摇杆        →
```

**[重难点]** 摇杆只负责产出 **`Vector2 direction`**；**怎么移动角色** 仍由 [01_角色控制器.md](01_角色控制器.md) 的 Move 逻辑负责。换输入源不应重写移动代码。

**[易错点]** Lab_Gameplay_01 要求：**只动摇杆、键盘不碰也能走** → 说明移动脚本读的是摇杆，不是 `Input.GetAxis`。

## 三、EasyTouch Joystick 控件（⭐⭐⭐）

### 3.1 创建与配置

| 步骤 | 操作 | 频率 |
|:---:|------|------|
| 1 | Hierarchy → Canvas 下添加 EasyTouch **Joystick** | ⭐⭐⭐ |
| 2 | 设置 **Position**（如左下角 Fixed） | ⭐⭐⭐ |
| 3 | 配置 **Background / Thumb** 图片 | ⭐⭐ |
| 4 | **Axis Options**：Horizontal (X) / Vertical (Y) | ⭐⭐⭐ |
| 5 | 移动脚本中读取 Joystick 输出 | ⭐⭐⭐ |

### 3.2 Axis Options

| 轴 | 含义 | 典型映射 |
|----|------|----------|
| **X（Horizontal）** | 水平方向 | 左右移动 |
| **Y（Vertical）** | 垂直方向 | 前后移动 |

### 3.3 读取输入

| API / 方式 | 说明 | 频率 |
|------------|------|------|
| `GetAxisValue()` | 获取 0~1 范围力度（具体 API 以插件版本为准） | ⭐⭐⭐ |
| Joystick 实例 `.axisX` / `.axisY` | 部分版本直接读轴向量 | ⭐⭐⭐ |
| 合成 `Vector2` | `(axisX, axisY)` 作为 direction | ⭐⭐⭐ |

```csharp
// 示意：以 EasyTouch Joystick 为例（API 名随版本略有差异，以 Inspector 文档为准）
public class JoystickInput : MonoBehaviour
{
    public ETCJoystick joystick;  // EasyTouch Joystick 组件

    public Vector2 GetDirection()
    {
        return new Vector2(joystick.axisX.axisValue, joystick.axisY.axisValue);
    }
}
```

**[教材精华·陈俊宇]** `GetAxisValue()` 取 **0~1 力度**，用于区分「轻推慢跑 / 推满快跑」；Lab L1 可先当开关量用（有输入就走）。

## 四、Dead Zone 与归一化（⭐⭐⭐）

### 4.1 Dead Zone（死区）

| 概念 | 说明 | 频率 |
|------|------|------|
| **Dead Zone** | 输入 magnitude 低于阈值时视为 0 | ⭐⭐⭐ |
| **为什么** | 手指微抖、摇杆回弹不完全 → 角色漂移 | ⭐⭐⭐ |
| **典型值** | 0.1 ~ 0.2（按手感调） | ⭐⭐ |

```csharp
Vector2 ApplyDeadZone(Vector2 input, float deadZone = 0.15f)
{
    if (input.magnitude < deadZone)
        return Vector2.zero;
    return input;
}
```

**[面试常考·Lab_Gameplay_01]** Dead zone 是什么？→ 过滤低于阈值的微小输入，防止角色静止时漂移。

### 4.2 对角线速度修正

```
摇杆推到右下角：(1, 1)
magnitude = √2 ≈ 1.414  →  比单轴推满更快！
```

| 问题 | 解决 | 频率 |
|------|------|------|
| 对角线更快 | `if (dir.sqrMagnitude > 1f) dir.Normalize()` | ⭐⭐⭐ |
| 或始终 Normalize | 只要方向不要力度时 | ⭐⭐ |

**[教材精华·宣雨松/Exam02]** 与键盘 `Input.GetAxis` 同理：合成向量后需 Normalize，否则对角线超速。

## 五、输入平滑（⭐⭐）

| 方法 | 说明 | 频率 |
|------|------|------|
| `Mathf.Lerp(current, target, t)` | 线性插值逼近目标输入 | ⭐⭐ |
| `Vector2.SmoothDamp(...)` | 带惯性的平滑，手感更自然 | ⭐⭐ |
| 插件内置 Smooth | 部分 Joystick 自带 Smooth 参数 | ⭐⭐ |

```csharp
Vector2 rawInput;
Vector2 smoothedInput;
public float smoothTime = 0.1f;
Vector2 velocity;

void Update()
{
    rawInput = GetJoystickDirection();
    rawInput = ApplyDeadZone(rawInput);
    smoothedInput = Vector2.SmoothDamp(smoothedInput, rawInput, ref velocity, smoothTime);
}
```

**[教材精华·吴亚峰]** 平滑不是 L1 必须，但 P+1 / 面试会问；轻推摇杆时 SmoothDamp 可减少「瞬移感」。

## 六、对接 CharacterController 移动（⭐⭐⭐）

### 6.1 基本映射

```csharp
public class PlayerJoystickMove : MonoBehaviour
{
    CharacterController controller;
    public JoystickInput joystickInput;
    public float moveSpeed = 5f;
    public Transform cameraTransform;

    void Awake()
    {
        controller = GetComponent<CharacterController>();
        if (cameraTransform == null && Camera.main != null)
            cameraTransform = Camera.main.transform;
    }

    void Update()
    {
        Vector2 input = joystickInput.GetDirection();
        input = ApplyDeadZone(input);
        if (input.sqrMagnitude > 1f) input.Normalize();

        // 相机相对
        Vector3 camForward = cameraTransform.forward;
        Vector3 camRight = cameraTransform.right;
        camForward.y = 0f; camRight.y = 0f;
        camForward.Normalize(); camRight.Normalize();

        Vector3 moveDir = camForward * input.y + camRight * input.x;
        controller.Move(moveDir * moveSpeed * Time.deltaTime);

        // 可选：朝向跟随
        if (moveDir.sqrMagnitude > 0.001f)
            transform.rotation = Quaternion.LookRotation(moveDir);
    }

    Vector2 ApplyDeadZone(Vector2 v, float dz = 0.15f)
        => v.magnitude < dz ? Vector2.zero : v;
}
```

### 6.2 输入抽象（键盘 + 摇杆共用）

```csharp
Vector2 GetMoveInput()
{
#if UNITY_EDITOR || UNITY_STANDALONE
    // PC 调试备用
    return new Vector2(Input.GetAxis("Horizontal"), Input.GetAxis("Vertical"));
#else
    return joystickInput.GetDirection();
#endif
}
```

**[重难点]** 移动函数只认 `Vector2`，不关心输入来自键盘还是摇杆 → **Lab 与 Exam04 升级** 的核心工程思维。

## 十、自写 UGUI 摇杆（备选 · ⭐⭐）

EasyTouch 不可用时可自绘，Lab 允许 **「EasyTouch 或自写 UI 摇杆」**。

| 步骤 | 操作 | 频率 |
|:---:|------|------|
| 1 | Canvas 下建 Background（Image）+ Handle（Image） | ⭐⭐ |
| 2 | 脚本实现 `IPointerDownHandler` / `IDragHandler` / `IPointerUpHandler` | ⭐⭐ |
| 3 | 拖动手柄限制在 Background 半径内 | ⭐⭐ |
| 4 | 输出 `(handle.localPosition / radius)` 作为 Vector2 | ⭐⭐ |
| 5 | 松手 Reset 手柄到中心 | ⭐⭐ |

```csharp
public Vector2 Direction { get; private set; }

public void OnDrag(PointerEventData eventData)
{
    Vector2 localPos;
    RectTransformUtility.ScreenPointToLocalPointInRectangle(
        background, eventData.position, eventData.pressEventCamera, out localPos);

    localPos = Vector2.ClampMagnitude(localPos, radius);
    handle.anchoredPosition = localPos;
    Direction = localPos / radius;
}

public void OnPointerUp(PointerEventData eventData)
{
    handle.anchoredPosition = Vector2.zero;
    Direction = Vector2.zero;
}
```

**[教材精华·王磊]** 自写摇杆本质是 UGUI 事件 + RectTransform 数学；理解后不被插件绑死。

## 十二、注意要点与最佳实践

### 常见陷阱

| 易错点 | 说明 |
|--------|------|
| 无 EventSystem | 摇杆无响应 |
| 未归一化对角线 | 斜向移动更快 |
| 无 dead zone | 角色漂移 |
| 输入与移动耦合 | 换键盘/摇杆要改一堆代码 |
| 每帧 GetComponent | 应缓存 Joystick 引用 |
| 摇杆直接 Move | 应经 direction 抽象层 |
| PC 不测摇杆 | Editor 用鼠标模拟触摸或保留键盘 fallback |

### 最佳实践

- **输入层** 产出 Vector2 → **移动层** 只认 Vector2
- 必做 dead zone + 对角线 Normalize
- 第三人称用相机相对移动（见 01 文档）
- Import 插件后先测 Joystick 能否拖动，再写移动
- Lab 验收：只动摇杆能走、2 状态能切（见 03_状态机.md）

## 十六、教材 vs 官方 · 对照速查

| 知识点 | 教材常见写法 | 官方/工程推荐 |
