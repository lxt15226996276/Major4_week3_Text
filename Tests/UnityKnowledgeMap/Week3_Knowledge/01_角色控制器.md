## 一、类与组件关系（官方标准 · 必背）

```
Component
└── CharacterController（运动学角色碰撞体）
        ├── 内置 Capsule 碰撞形状（非 Rigidbody）
        ├── Move(offset) → CollisionFlags
        ├── isGrounded（bool）
        └── 属性：height / radius / center / stepOffset / slopeLimit

（常见搭配）
GameObject（Player）
├── CharacterController   ← 位移与碰撞
├── Animator              ← 动画状态机（Apply Root Motion 通常关）
└── PlayerController.cs   ← 读输入 → Move → Set 参数
```

**[重难点]** `CharacterController` **不是** Rigidbody：不受力、不自动响应物理碰撞，位移**完全由脚本 `Move()` 驱动**；引擎只帮你做**胶囊体与场景的碰撞检测与滑动**。

**[易错点]** Exam03/04 用 `transform.Translate` 能跑，但会**穿墙**；升级到 CharacterController 是 U11 核心考点。

## 二、CharacterController 组件与 Inspector（⭐⭐⭐）

### 2.1 组件本质

| 项目 | 内容 |
|------|------|
| **是什么** | 专为**人形/角色**设计的运动学碰撞体，胶囊形状 |
| **官方** | [Manual: Character Controller](https://docs.unity3d.com/Manual/class-CharacterController.html) · [API: CharacterController](https://docs.unity3d.com/ScriptReference/CharacterController.html) |
| **命名空间** | `UnityEngine` |
| **依赖** | **不需要** Rigidbody、Collider（自带胶囊） |

### 2.2 核心 Inspector 属性

| 属性 | 说明 | 频率 |
|------|------|------|
| **Height** | 胶囊体总高度 | ⭐⭐⭐ |
| **Radius** | 胶囊体半径（水平碰撞范围） | ⭐⭐⭐ |
| **Center** | 胶囊体相对 Transform 的偏移 | ⭐⭐ |
| **Step Offset** | 可自动跨过的台阶高度（默认 0.3） | ⭐⭐⭐ |
| **Slope Limit** | 可行走的最大坡度角（默认 45°） | ⭐⭐ |
| **Skin Width** | 与表面保持的微小距离，防抖动 | ⭐ |

**[教材精华·王磊/宣雨松]** 角色模型导入后，先对齐 **Height/Center** 使胶囊包住角色，否则 isGrounded 误判或「悬空/陷地」。

**[易错点]**
- Step Offset 大于 Height → Unity 会自动限制，但台阶仍可能过不去
- Radius 过大 → 卡窄门、卡墙角

## 三、Move 与 CollisionFlags（⭐⭐⭐）

### 3.1 Move API

| API | 说明 | 频率 |
|-----|------|------|
| `Move(Vector3 motion)` | 尝试移动 motion 距离，返回碰撞标志 | ⭐⭐⭐ |
| `SimpleMove(Vector3 speed)` | 按**速度**移动并**自动加重力**（少用，不够灵活） | ⭐ |
| `isGrounded` | 上一帧 Move 后是否接地 | ⭐⭐⭐ |

```csharp
public class PlayerMove : MonoBehaviour
{
    CharacterController controller;
    public float moveSpeed = 5f;

    void Awake()
    {
        controller = GetComponent<CharacterController>();  // ⭐⭐⭐ Awake 缓存
    }

    void Update()
    {
        float h = Input.GetAxis("Horizontal");
        float v = Input.GetAxis("Vertical");
        Vector3 move = new Vector3(h, 0f, v);

        // 必须乘 deltaTime — Move 参数是「本帧位移量」而非速度
        controller.Move(move * moveSpeed * Time.deltaTime);
    }
}
```

### 3.2 CollisionFlags

| 标志 | 含义 | 频率 |
|------|------|------|
| **None** | 无碰撞 | ⭐⭐ |
| **Sides** | 碰到侧面（墙） | ⭐⭐⭐ |
| **Above** | 碰到上方（天花板） | ⭐⭐ |
| **Below** | 碰到下方（地面） | ⭐⭐⭐ |

```csharp
CollisionFlags flags = controller.Move(totalMotion);
if ((flags & CollisionFlags.Sides) != 0)
{
    // 撞墙，可做音效/特效
}
```

**[重难点]** `Move()` 返回值是**本帧**碰撞结果；`isGrounded` 是**属性**，每帧 Move 后更新。跳跃逻辑应依赖 `isGrounded`，而非自己射线检测（初学阶段）。

**[易错点]**
- `Move(speed * direction)` 漏乘 `Time.deltaTime` → 帧率越高走得越快
- 一帧内多次 `Move` 拆分水平/垂直可以，但顺序影响 isGrounded 判断

## 四、重力与跳跃（⭐⭐⭐）

### 4.1 重力模型

CharacterController **不会自动施加重力**，必须脚本累积垂直速度：

```csharp
public class PlayerJump : MonoBehaviour
{
    CharacterController controller;
    public float moveSpeed = 5f;
    public float jumpForce = 5f;
    public float gravity = -9.81f;

    float velocityY;  // 垂直速度累积

    void Awake() => controller = GetComponent<CharacterController>();

    void Update()
    {
        // ── 水平 ──
        float h = Input.GetAxis("Horizontal");
        float v = Input.GetAxis("Vertical");
        Vector3 horizontal = new Vector3(h, 0f, v) * moveSpeed;

        // ── 跳跃 ──
        if (controller.isGrounded)
        {
            velocityY = -2f;  // 小负值贴地，防浮空抖动
            if (Input.GetButtonDown("Jump"))
                velocityY = jumpForce;
        }

        // ── 重力累积 ──
        velocityY += gravity * Time.deltaTime;

        // ── 合并位移 ──
        Vector3 motion = horizontal + Vector3.up * velocityY;
        controller.Move(motion * Time.deltaTime);
    }
}
```

### 4.2 跳跃关键点

| 步骤 | 说明 | 频率 |
|:---:|------|------|
| 1 | 检测跳跃输入（`GetButtonDown("Jump")`） | ⭐⭐⭐ |
| 2 | **必须** `isGrounded == true` 才允许跳 | ⭐⭐⭐ |
| 3 | 给 `velocityY` 赋正值（jumpForce） | ⭐⭐⭐ |
| 4 | 空中时只累积重力，不响应跳跃 | ⭐⭐⭐ |
| 5 | 可选：`animator.SetTrigger("Jump")` | ⭐⭐ |

**[教材精华·杜亚南/陈俊宇]** 跳跃 = 输入 + isGrounded 双条件；只写输入不写 isGrounded → **空中连跳**。

**[易错点]**
- 在 `isGrounded` 分支外直接 `velocityY = jumpForce` → 空中连跳
- 地面时 `velocityY = 0` 而非小负值 → 有时 isGrounded 抖动

## 五、相机相对移动（⭐⭐⭐）

第三人称/RPG 常见：**W 永远朝相机前方走**，而非角色本地 Z 轴。

```csharp
Transform camTransform = Camera.main.transform;

Vector3 camForward = camTransform.forward;
Vector3 camRight = camTransform.right;
camForward.y = 0f;
camRight.y = 0f;
camForward.Normalize();
camRight.Normalize();

Vector3 moveDir = camForward * v + camRight * h;
controller.Move(moveDir * moveSpeed * Time.deltaTime);

// 可选：角色朝向跟随移动方向
if (moveDir.sqrMagnitude > 0.001f)
{
    transform.rotation = Quaternion.LookRotation(moveDir);
}
```

**[教材精华·吴亚峰/马遥]** Exam03 用 `Space.Self` 坦克式移动；RPG 第三人称应转**相机相对**，与 W4 主场景操作一致。

**[重难点]** 必须**清零 Y 分量**再 Normalize，否则相机俯视时角色往地下/天上走。

## 六、下蹲与胶囊体动态调参（⭐⭐）

| 步骤 | 操作 | 频率 |
|:---:|------|------|
| 1 | 检测下蹲输入（如 Left Ctrl） | ⭐⭐ |
| 2 | `controller.height = crouchHeight` | ⭐⭐ |
| 3 | 同步改 `controller.center`（保持脚底不动） | ⭐⭐ |
| 4 | 播放下蹲/站立动画 | ⭐⭐ |

```csharp
void ApplyCrouch(bool crouching)
{
    if (crouching)
    {
        controller.height = 1f;
        controller.center = new Vector3(0, 0.5f, 0);
    }
    else
    {
        controller.height = 2f;
        controller.center = new Vector3(0, 1f, 0);
    }
}
```

**[易错点]** 只改 Height 不改 Center → 角色「陷进地面」或「悬空」。

## 七、与 Animator 联动（⭐⭐⭐）

| 脚本数据 | Animator 参数 | 用途 | 频率 |
|----------|--------------|------|------|
| 水平速度 magnitude | `SetFloat("Speed", speed)` | Idle ↔ Run BlendTree | ⭐⭐⭐ |
| 是否在地面 | `SetBool("IsGrounded", ...)` | 空中/落地状态 | ⭐⭐ |
| 跳跃输入 | `SetTrigger("Jump")` | 跳跃动画 | ⭐⭐⭐ |
| 是否在移动 | `SetBool("IsWalking", isMove)` | Exam04 走跑切换 | ⭐⭐⭐ |

```csharp
float speed = new Vector3(h, 0f, v).magnitude;
animator.SetFloat("Speed", speed);
animator.SetBool("IsWalking", speed > 0.01f);
```

**[重难点] Root Motion 与脚本 Move 的分工**

| Apply Root Motion | 水平位移谁写 | 适用 |
|-------------------|-------------|------|
| **关闭**（主流） | 脚本 `Move()` | RPG、射击、Lab_Gameplay |
| **开启** | 动画驱动位移 | 部分动作游戏、精准步伐 |

**[糟粕剔除]** Root Motion 开 + 脚本仍 `Move` 水平位移 → **双倍移动**；与 Week2 Animator 文档一致，选一个主控。

## 八、Translate / Rigidbody / CharacterController 选型（⭐⭐⭐）

| 对比项 | transform.Translate | Rigidbody | CharacterController |
|--------|---------------------|-----------|---------------------|
| 碰撞 | ❌ 穿墙 | ✅ 物理碰撞 | ✅ 胶囊碰撞+滑动 |
| 重力 | ❌ 需手写 | ✅ 自动 | ❌ 需手写 |
| 推物体 | ❌ | ✅ | ❌（运动学） |
| 台阶/斜面 | ❌ | 看物理设置 | ✅ Step Offset / Slope Limit |
| 手感 | 直接 | 偏物理真实 | **可控、稳定** |
| 典型 | Exam02 练手 | 球类、车、布娃娃 | **RPG/动作角色** |
| 课程 | Exam02/03 | 了解即可 | **U11 · Exam04 升级** |

**[教材精华·Gregory/宣雨松]** 运动学角色（CC）与动力学刚体是引擎两套碰撞哲学；RPG 要**稳定可控** → CharacterController；需要被炸飞、推箱子 → Rigidbody。

**[面试常考]** CharacterController vs Rigidbody：CC 不依赖刚体、不受力、Move 驱动；RB 物理驱动、可交互力。

## 九、从 Exam Translate 升级到 CharacterController

| 阶段 | 写法 | 问题 |
|------|------|------|
| Exam02/03 | `transform.Translate(..., Space.Self)` | 穿墙、无台阶 |
| Exam04 过渡 | Translate + Animator IsWalking | 动画有了，碰撞仍缺 |
| **U11 目标** | `controller.Move(...)` + 重力 + isGrounded | 完整角色控制器 |

```csharp
// Exam04 坦克式（待升级）
transform.Translate(new Vector3(0, 0, v) * moveSpeed * Time.deltaTime, Space.Self);
transform.Rotate(0, h * turnSpeed * Time.deltaTime, 0);

// U11 升级方向
Vector3 move = transform.forward * v + transform.right * h; // 或相机相对
// + 重力 velocityY
controller.Move((move * moveSpeed + Vector3.up * velocityY) * Time.deltaTime);
```

## 十、注意要点与最佳实践

### 常见陷阱

| 易错点 | 说明 |
|--------|------|
| 漏乘 Time.deltaTime | Move 参数是位移量，不是速度 |
| 不手写重力 | 角色空中不下落 |
| 跳跃不检 isGrounded | 空中连跳 |
| CC + Rigidbody 同挂 | 双碰撞体冲突 |
| Root Motion + Move 双写 | 位移加倍 |
| Height/Center 不匹配模型 | isGrounded 误判 |
| Update GetComponent | 应 Awake 缓存 |

### 最佳实践

- 3D 可控角色 → CharacterController + 脚本 Move
- 水平与垂直速度分开管理，最后合并一次 Move（或分两次 Move）
- 地面时 velocityY 设小负值（-2f）而非 0
- 移动速度/状态同步 Animator 参数，不直接 Play Clip
- Awake 缓存 CharacterController、Animator、Camera 引用
- 与 Lab_Gameplay_01 衔接：键盘版先过关，再换摇杆输入源（见 02_摇杆插件.md）
