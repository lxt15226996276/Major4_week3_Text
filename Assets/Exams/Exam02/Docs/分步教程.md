### 3.1 AccountData 单例 + Dictionary

#### 本段思路

1. **AuthManager** 上挂 **唯一** `AccountData`，`Awake` 做单例 + `DontDestroyOnLoad`，进选服后字典还在。  
2. **Register**：Trim → 判空 → 写入 `Dictionary` → Log 成功（卷面「注册存字典」）。  
3. **TryLogin**：`TryGetValue` 查键，再比对密码（无键不抛异常）。  
4. **3b/3c** 才在 `LoginUIController` 读 InputField 调这两个方法。

#### 场景操作

|:----:|------|

| 2 | 挂 `Scripts/AccountData.cs` |

#### 🎯 P-L1 带注释范本（整文件对照敲）

```csharp
using System.Collections.Generic;
using UnityEngine;

namespace Exam.Exam02
{
    /// <summary>
    /// Exam02 账号字典：键=账号，值=密码（运行时内存，非网络）
    /// </summary>
    public class AccountData : MonoBehaviour
    {
        // --- 单例：全场景唯一，UI 通过 AccountData.Instance 调 Register/TryLogin ---
        public static AccountData Instance { get; private set; }

        // --- 数据：账号(string) → 密码(string)，考试要求必须用 Dictionary ---
        private readonly Dictionary<string, string> accountDic = new Dictionary<string, string>();

        private void Awake()
        {
            // 若场景里误拖了两个 AuthManager，销毁多余的，保证 Instance 唯一
            if (Instance != null && Instance != this)
            {
                Destroy(gameObject);
                return;
            }
            Instance = this;
            // 进 Exam02_ServerScene 后仍保留已注册账号
            DontDestroyOnLoad(gameObject);
        }

        /// <summary>
        /// 注册：Trim 后写入字典。空账号/密码返回 false。
        /// </summary>
        public bool Register(string account, string password)
        {
            account = account?.Trim();
            password = password?.Trim();
            if (string.IsNullOrEmpty(account) || string.IsNullOrEmpty(password))
                return false;

            accountDic[account] = password;  // 同账号再次注册会覆盖密码（P+1 可改重复提示）
            Debug.Log($"[注册成功] 账号={account}，当前总数={accountDic.Count}");
            return true;
        }

        /// <summary>
        /// 登录：字典里有该账号且密码一致返回 true。
        /// </summary>
        public bool TryLogin(string account, string password)
        {
            account = account?.Trim();
            password = password?.Trim();
            // TryGetValue：无此账号时 stored 无效且返回 false，不会 KeyNotFoundException
            return accountDic.TryGetValue(account, out string stored) && stored == password;
        }

        /// <summary>当前已注册账号数量（自测「至少 3 组」用）</summary>
        public int Count => accountDic.Count;
    }
}
```

#### 逐行讲解（需注释的行）

| 行/块 | 含义 |
|-------|------|
| `Instance` + Awake 里 Destroy 重复 | 单例模式；多实例时只留一个 |
| `DontDestroyOnLoad` | 切场景不销毁，否则进选服字典清空 |
| `accountDic` | 试卷考点：键值存储，禁止三组 if 写死 |
| `accountDic[account]=password` | 写入；重复键会覆盖（P+1 用 ContainsKey 提示） |
| `TryGetValue(..., out stored)` | 安全查键再比密码 |

**B本步 · 原理**

**B本步 · 带注释对照代码**（对照阅读 · P+1 #1）

```csharp
/// <summary>注册：重复账号不覆盖，通过 msg 告知 UI</summary>
public bool TryRegister(string account, string password, out string message)
{
    account = account?.Trim();
    password = password?.Trim();
    message = "";
    if (string.IsNullOrEmpty(account) || string.IsNullOrEmpty(password))
    {
        message = "账号或密码不能为空";
        return false;
    }
    // ContainsKey：重复注册时给出明确反馈，而不是覆盖旧密码
    if (accountDic.ContainsKey(account))
    {
        message = "账号已存在";
        return false;
    }
    accountDic.Add(account, password);
    message = "注册成功";
    return true;
}
```

```csharp
            zoneColorImage.color = zoneColors[index];
```

```csharp
/// Exam02 选服：Dropdown 改色 + BtnEnter 异步进对应 Game 场景（第 5～6 条）
public class ServerManager : MonoBehaviour
{
    // --- UI 引用（Inspector 拖）---
    [SerializeField] private Dropdown serverDropDown;
    [SerializeField] private Button btnEnter;
    [SerializeField] private GameObject loadingPanel;
    [SerializeField] private Slider progressSlider;
    [SerializeField] private Text progressText;

    // index 与 Dropdown Options 0/1/2 同序
    private readonly string[] sceneNames = { "Exam02_Game1", "Exam02_Game2", "Exam02_Game3" };

    void Awake()
    {
        if (loadingPanel != null)
            loadingPanel.SetActive(false);
    }

    void Start()
    {
        serverDropDown.onValueChanged.AddListener(SelectedChanged);
        btnEnter.onClick.AddListener(() => StartCoroutine(LoadGameScene(serverDropDown.value)));
        SelectedChanged(serverDropDown.value);
    }

    /// 思路：AsyncOperation → 0~0.9 刷 Slider → allow 激活切场景
    IEnumerator LoadGameScene(int index)
    {
        loadingPanel.SetActive(true);
        progressSlider.value = 0f;

        AsyncOperation op = SceneManager.LoadSceneAsync(sceneNames[index]);
        op.allowSceneActivation = false;

        while (op.progress < 0.9f)
        {
            float p = op.progress / 0.9f;
            progressSlider.value = p;
            if (progressText != null)
                progressText.text = $"加载中...{(int)(p * 100)}%";
            yield return null;
        }

        progressSlider.value = 1f;
        yield return new WaitForSeconds(0.3f);
        op.allowSceneActivation = true;
    }
}
```

#### 逐行讲解表

| 行/块 | 含义 |
|-------|------|
| `sceneNames[]` | 与 Dropdown 0/1/2 一一对应；Build 名须一致 |
| `allowSceneActivation = false` | 资源就绪但不立即卸载 ServerScene |
| `op.progress / 0.9f` | Unity progress 上限约 0.9，不除则 Slider 填不满 |
| `yield return null` | 每帧让出主线程，Slider 才能重绘 |
| `allowSceneActivation = true` | 最后 0.1 才真正切换 |
| `btnEnter.onClick` | 第 5 条：必须异步，禁止同步 `LoadScene` |

| API | 含义 |

| `allowSceneActivation` | false=卡在 0.9；true=完成切换 |
